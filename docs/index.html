How it works

Practical principles, built into the product.

People stay in control

AI tools should serve people. Your library and clipboard stay local unless you deliberately export or share something.

Loads only what is needed

Agents receive short resource summaries first, then fetch full content by id when a task actually needs it.

Works from the keyboard

Search clipboard history, switch tabs, and paste recent items without leaving the app you are working in.

Save reusable material

Keep prompts, Skills, MCP references, and knowledge in one place. Enable each item independently and see its real Agent usage count.

Share only what you choose

Select individual resources for a portable JSON bundle. Stable IDs and content matching prevent repeated imports on another computer.

Let AI clean up the library

Fetch everything at once or by category. Usage metadata and duplicate groups help an Agent explain what is redundant or never used.

Hear when work finishes

DingDong plays the selected sound and marks the menu bar icon when an agent finishes or needs your attention.

Finish alerts

Know when an Agent is ready.

DingDong notifies you when an Agent finishes, gets blocked, or needs a decision, so you can continue other work without repeatedly checking its status.

Choose your own finish sound

Pick any built-in sound or import your own audio file. Every connected agent uses the sound selected in DingDong Settings.

Choose a sound to preview it

Built-in MCP bridge

One MCP connection for every local agent.

DingDong serves only the prompts, skills, MCP references, and knowledge you have enabled. Manage the switches once instead of repeating the same setup in every agent.

Agent config

Add this MCP server to Codex or Claude Code after installing DingDong.

  • Install DingDong in /Applications.
  • Register the built-in dingdong-mcp binary in your agent MCP config.
  • Keep prompts, skills, and MCP references in DingDong instead of duplicating them in every agent.
{
  "mcpServers": {
    "dingdong": {
      "command": "/Applications/DingDong.app/Contents/MCP/bundle/bin/dingdong_mcp"
    }
  }
}
[mcp_servers.dingdong]
command = "/Applications/DingDong.app/Contents/MCP/bundle/bin/dingdong_mcp"
dingdong_bridge

Start here. Returns task-scoped summaries, recommended resources, and privacy rules.

dingdong_search_assets

Search prompts, skills, MCP references, notes, and other resources by query and type.

dingdong_get_asset

Fetch one resource by id. Summary mode hides heavy content; full mode is explicit.

dingdong_load_skill

Load a selected skill resource only after an agent decides it is relevant.

dingdong_recommend_mcp

Find MCP server references that match the current task.

dingdong_install_native_mcp

Dry-run or install a stored MCP reference into Codex or Claude Code config.

dingdong_notify

Trigger the DingDong sound, unread badge, and menu bar notification icon.
